:memo: Bug Report Requirements
Each bug report must include the following:
Title
Description
Steps to Reproduce
Expected Result
Actual Result
Severity
Priority
Include screenshots wherever applicable
 :small_orange_diamond: Categorize the bugs (UI, functionality, performance, etc.)
Notes & Guidance
Ensure your bugs are clear, concise, and reproducible
Always verify an issue before reporting it
Don’t ignore edge cases — try decimals, large inputs, invalid values, etc.
Think like a user: how would this bug affect the experience?
:package: Deliverables
10–15 well-documented bug reports using the standard format
A brief write-up (approx. 200 words in Google Docs) explaining your testing approach, strategy, and any tools used
